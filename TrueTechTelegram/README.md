# TrueTechTelegram

Telegram-бот на `python` и `aiogram`, который за 7 быстрых шагов собирает минимальный профиль пользователя, показывает персональный маршрут и дальше ведет пользователя по чатовым модулям курса.

## Быстрый запуск

1. Создайте `.env` из примера:

```bash
cp .env.example .env
```

2. Укажите в `.env` токен Telegram-бота:

```bash
BOT_TOKEN=1234567890:your_real_token
DATABASE_PATH=data/bot.sqlite3
```

3. Запустите контейнер:

```bash
docker compose up --build
```

Mini app в текущей версии отключен: основной сценарий проходит внутри Telegram через одно центральное сообщение и inline-кнопки. Исключение — галерея фотосертификатов: бот удаляет центральное сообщение, присылает фото с пагинацией, а при возврате восстанавливает меню новым сообщением.

## Команды бота

- `/start` — запуск теста и создание персонального маршрута.
- `/restart` — сброс текущего теста и старт заново.
- `/certificates` — список загруженных сертификатов через редактирование центрального сообщения.

## Что уже есть

- 7 быстрых вопросов: цель, конкретный навык/специальность, возраст, желаемый результат, уровень/опыт/навыки, доступное время и ограничения.
- Ветвящиеся вопросы: следующие шаги и варианты меняются по цели, интересу, возрасту и выбранному времени.
- В первой развилке есть вариант `Не знаю, что выбрать`: для него бот подбирает короткий список специальностей и маршрут выбора направления.
- Второй шаг показывает динамический TOP-10 конкретных навыков или специальностей под выбранную цель.
- TOP-10 считается по реальным запросам пользователей: выбранным кнопкам и ручному вводу на втором шаге.
- Если нужного навыка или специальности нет в TOP-10, пользователь вводит свой вариант сообщением, и он сразу сохраняется.
- Inline-кнопки с эмоциональными подписями по две в ряд и ручной ввод с клавиатуры на каждом вопросе.
- В вопросе про опыт клавиатура идет в один столбец, чтобы длинные варианты не ломали разметку.
- В мультивыборе свой текст сохраняется без кнопки подтверждения; удалить его можно кнопкой `Очистить свои варианты`.
- На каждом шаге после первого есть кнопка `Назад`.
- На 7-м вопросе мультивыбор: можно отметить несколько ограничений и нажать `Готово`.
- Редактирование текущего сообщения после каждого ответа бота.
- После онбординга появляется центральное меню маршрута: из него открывается текущий модуль, сертификаты и загрузка сертификатов.
- Индикатор прогресса на каждом шаге.
- Промежуточный экран построения маршрута на 2-3 секунды.
- Краткий портрет пользователя в результате: цель, навык/специальность, возраст, ожидаемый результат, опыт/навыки, время и ограничения.
- После онбординга бот показывает сообщение о готовом курсе и кнопку `Начать прохождение курса`.
- Курс идет через редактирование центрального сообщения: модуль открывается из меню, а из модуля можно вернуться назад в меню.
- Пользователь может отметить модуль пройденным кнопкой или текстом `модуль пройден`.
- Под каждым модулем есть быстрый фидбек: полезно, сложно, слишком просто, не подошло.
- После завершения всех модулей бот выдает достижение и сертификат в стиле «Прогрессоров» с кодом подтверждения.
- Пользователь может загружать свои сертификаты в бота: PDF, изображения и другие файлы сохраняются в локальное хранилище `data/certificates`.
- Раздел `Сертификаты` открывается одной кнопкой. При просмотре фотосертификатов бот удаляет центральное текстовое сообщение, присылает фото с пагинацией по 5 штук, а при возврате удаляет фото и восстанавливает центральное меню.
- SQLite-база для пользователей, сессий теста, ответов и аналитических событий.
- Заглушка аналитики `track_event`, которая пишет события в лог и таблицу `analytics_events`.

## Методология вопросов

Сценарий использует воронку минимального профиля:

1. `Зачем учиться` — цель: карьера, навык или учеба.
2. `Что изучать` — TOP-10 популярных сейчас конкретных навыков или специальностей под выбранную цель.
3. `Кто пользователь` — возрастной контекст для тона и нагрузки.
4. `Что получить на выходе` — ближайшая понятная точка: база, практика, первый результат, учебный проект или выбор профессии.
5. `Откуда стартуем` — текущий уровень, опыт и навыки в выбранной сфере.
6. `Какой ресурс есть` — реальное время на обучение.
7. `Что мешает` — ограничения и предпочтения в формате обучения, через мультивыбор.

Такой порядок сначала определяет направление, затем сужает маршрут до реалистичного формата. Поэтому следующие вопросы и варианты ответов адаптируются по уже выбранным данным.

TOP-10 строится из базы данных по фактическим ответам пользователей. Seed-список в коде нужен только для холодного старта, пока пользовательских запросов еще недостаточно; дальше список переупорядочивается и пополняется тем, что люди реально выбирают или вводят.

## Данные

По умолчанию база создается в `data/bot.sqlite3`. В Docker Compose папка `./data` подключена как volume, поэтому ответы пользователей сохраняются между перезапусками контейнера.

Основные таблицы:

- `users` — Telegram-профиль пользователя.
- `quiz_sessions` — прохождение теста, текущий шаг, финальный профиль и результат.
- `quiz_answers` — каждый ответ пользователя с вопросом, вариантом и источником: кнопка или ручной ввод.
- `analytics_events` — события сценария: `quiz_start`, `question_answered`, `offer_timer_started`, `result_shown` и другие.
- `course_sessions` — прохождение чатового курса, текущий модуль, маршрут и код сертификата.
- `course_module_events` — отметки прохождения модулей и обратная связь по материалам.
- `user_certificates` — загруженные пользователем сертификаты и локальные пути к файлам.

## Курс после онбординга

Сейчас материалы в курсе являются временным контентом. Они описывают тип будущего материала, источник, примерное время и действие пользователя. Это позволяет демонстрировать механику маршрута без подключения реального каталога бесплатных русскоязычных ресурсов.

Базовый маршрут состоит из 4 модулей:

1. Старт и цель — пользователь понимает цель и стартовую точку.
2. База направления — короткая теория и проверка понимания.
3. Практика — пользователь делает маленький видимый результат.
4. Итог и следующий шаг — фиксация результата, достижение и сертификат.

## Идеи для развития

- Автозамена материалов по фидбеку: слишком сложно, слишком просто, не подошел формат.
- Еженедельный план с напоминаниями и мягким переносом дедлайнов.
- Экспорт маршрута в PDF-карту или карточку для приложения «Прогрессоры».
- Проверочные мини-квизы после модулей и выдача бейджей за навыки.
- Подбор реальных бесплатных русскоязычных материалов через каталог источников.
- Рекомендации карьерных ролей и учебных проектов после каждого этапа.
- Режим наставника: пользователь присылает работу, бот дает чек-лист обратной связи.
- Групповые челленджи и лидерборд по прогрессу внутри класса, вуза или команды.
- Интеграция с картой «Прогрессоров»: модули как точки маршрута, достижения как артефакты.
- Верификация загруженных сертификатов: извлечение названия, даты, платформы и статуса проверки.
