# Архитектурная заметка

Прототип устроен по слоям, близким к Clean Architecture. Основной продуктовый вход — Telegram-бот, Mini App используется как основной экран маршрута внутри Telegram.

## Domain

`roadmapBuilder.js` не знает про DOM, Telegram или хранение данных. Он принимает профиль и каталог, затем возвращает пользовательский срез базы знаний:

- нормализованный профиль;
- выбранный domain и track;
- ограничения маршрута по режиму;
- модули, разделы, темы и материалы;
- вычисленный прогресс и объяснение.

Позже сюда можно добавить LLM-нормализацию цели, но ссылки и материалы всё равно должны приходить из контролируемого каталога.

## Infrastructure

В Mini App `mockCatalog.js` сейчас заменяет backend и админку. В реальном сервисе этот слой станет API-клиентом:

- `GET /api/roadmaps/current`;
- `POST /api/roadmaps/generate`;
- `POST /api/progress/mark`;
- `POST /api/feedback`;
- `POST /api/materials/replace`.

## Presentation

`app.js` отвечает только за визуальное состояние Mini App. Он не содержит правил отбора материалов, поэтому UI можно переносить на React/Vite без изменения доменной логики.

## Bot

`bot/main.py` использует Telegram Bot API через long polling без внешних зависимостей. Бот отвечает за онбординг и передачу пользователя в Mini App через `web_app`-кнопку; построение маршрута и дальнейшая работа с прогрессом остаются в Mini App.
